# Day 022 - Build

**Date:** 2026-07-13
**Pillar:** Build

## Tidbit

This build made concrete the case the whole week-3 apparatus exists for, and building it surfaced a
failure mode I had not named. A review is supposed to be an independent second look, but a second check
only catches what the first one missed if the second checker actually does the thing the first one
skipped. Here the builder and the reviewer eyeballed the same secrets item the same way, so two controls
collapsed into one look taken twice. A secret typed into a node is the perfect illustration, because a
valid token gives no sign of where it lives, so "it runs" fooled both of them, and the send-back rate
scored the handoff a clean zero, indistinguishable from real prep. That clean zero is exactly why the
review miss rate had to exist: the number that catches the case the first number is blind to by
construction. Redundancy on paper is a single point of failure in practice unless the second look is
genuinely a different one.

## Shipped

`examples/a-rubber-stamped-approval.md` (v1): the second worked example, the false-approval counterpart to
Day 015. It traces one synthetic build, `feedback-form-to-crm`, with a secret typed into a node's
authorization header (a plain standard section 5 violation), through the loop the opposite way from Day
015. The builder marks the secrets item passing without opening the node, the reviewer rubber-stamps the
same item the same way, and the build is approved. The send-back rate scores the handoff a clean zero,
indistinguishable from real prep, because it counts the first-review outcome and that was Approve. Then a
spot-audit pulls the approved handoff (the sample is weighted toward approvals), the auditor opens the
node, finds the token, and overturns it: a miss, and specifically a false approval, the subset the review
miss rate counts and the send-back rate cannot see. It closes on the miss conversation (a real gap, open
the node next time, a habit to fix and not a charge) and an honest limit: one build, a single miss not a
pattern, caught only because it happened to be sampled. It makes concrete the counterfactual Day 015 named
and shows the week-3 audit loop catching the case it exists for.
