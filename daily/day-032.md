# Day 032 - Document

**Date:** 2026-07-23
**Pillar:** Document

## Tidbit

Seventeen files is where a set stops being a set and becomes a wall. Each piece was written to be findable
on its own, and none of them was written to be the first one somebody opens, which is how a folder that is
organized can still be unusable. The folders group by pillar, and that is right for the repo and useless to
a newcomer, because nobody arrives needing everything in one folder and nobody's job is called governance.
So the index orders the same pieces by when you need them and groups the doing by role, since most people
will only ever run one row of it. The design choice I want to keep is that it points and does not
summarize. A summary competes with its source and drifts from it, and an index that repeats content has to
be rewritten every time any piece changes, so pointing stays true longer. That matters because an index has
a failure mode the pieces do not: it goes stale silently, and a wrong pointer is worse than no pointer,
since an index is trusted. The file says that about itself, which is the only honest thing to do with a
document whose main risk is quietly lying later.

## Shipped

`sops/the-review-loop-in-order.md` (v1): the index that turns the review-side folder tree into a path. It
orders all seventeen pieces by use rather than by folder, in four passes: learn the loop (the week-2 map
for the shape, the standard for the bar, the checklist for the gate, then the week-3 map and spot-audit for
why the loop watches itself, then the week-4 map for what a pool costs), do your part (grouped by role, so
a builder, a reviewer, whoever assigns, whoever keeps the log, and an auditor each read one row), keep and
measure (the two records and the four numbers read off them), and see it run (the three worked examples,
with one line on what each shows). It names what is deliberately left out and why, and it closes on its own
limits: it goes stale where the pieces do not, it is a path and not a curriculum, reading order is not
doing order, and it cannot make anyone adopt any of this. Every path and version it cites was checked
against the tree, twenty-two paths and twenty-four version claims, zero mismatches. The README now points
at it, since an index nobody finds is not an index.
