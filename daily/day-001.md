# Day 001 - Build

**Date:** 2026-06-22
**Pillar:** Build

## Tidbit

What separates a dependable automation from a clever demo is convention, not cleverness:
clear naming, real error paths, only retrying steps that are safe to run twice, writes that
cannot create duplicates, keys limited to what they actually need, and enough logging to see
what happened. Reliability is a set of habits applied every time, not something you bolt on at
the end.

## Shipped

`standards/automation-standards.md` (v1): a tool-agnostic baseline every shared workflow
should meet, across seven areas (naming, error handling, retries, idempotency, secrets,
observability, documentation).
