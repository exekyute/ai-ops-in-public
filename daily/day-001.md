# Day 001 — Build

**Date:** 2026-06-22
**Pillar:** Build

## Tidbit

What separates a dependable automation from a clever demo is convention, not cleverness:
naming, error paths, retries on idempotent steps only, idempotent writes, scoped secrets,
and observability. Reliability is a set of habits applied every time, not a feature you add
at the end.

## Shipped

`standards/automation-standards.md` (v1): a tool-agnostic baseline every shared workflow
should meet, across seven areas (naming, error handling, retries, idempotency, secrets,
observability, documentation).
