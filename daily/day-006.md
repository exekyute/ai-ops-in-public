# Day 006 - Integrate

**Date:** 2026-06-27
**Pillar:** Integrate

## Tidbit

A week of separate artifacts is not yet a system, and the gap between the two is the seams.
Five documents that each make sense on their own can still leave a reader asking how they
connect. The integration work does not add content. It makes the order and the handoffs
explicit: what each piece hands to the next, and how the last one loops back to the first. Most
of the value in a set of parts sits in those joins, and the joins are the thing you have to
write down on purpose, because nobody sees them by reading the parts one at a time.

## Shipped

`integration/week-01-automation-lifecycle.md` (v1): a one-page map that puts this week's five
pieces in order as the stages a shared automation moves through, build it to a standard, review
it before it ships, get people using it, measure whether it works, and feed what you learn back
into the build. It names the handoff between each stage and the next, and the gaps a real
program would still need to close.
