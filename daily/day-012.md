# Day 012 - Analyze

**Date:** 2026-07-03
**Pillar:** Analyze

## Tidbit

The easy metric to add is the one the tool already tracks. The useful one this week was not in any
run log: whether a build arrives ready for review lives in the review records, the filled
checklists, not the execution history. That is worth noticing, because the number you most want
sometimes sits in a record you have to choose to keep. Defining it also forced me to say which stage
it measures. Exception rate is about a live build's runtime; send-back rate is about the handoff,
one step earlier, and calling that out keeps two health signals from being read as one. The trap I
had to write down is that this metric is only as honest as the review behind it: a reviewer who
waves everything through drives the send-back rate to zero and makes it meaningless. A number
inherits the integrity of the process that produces it, so a metric that can be gamed by going easy
is one you have to read next to the behavior it assumes.

## Shipped

`analytics/metric-definitions.md` taken to **v2** with a second metric, **Send-back rate**: of the
builds handed off for review in a period, the share a reviewer sends back on the first review rather
than approving. It is the measure of whether the builder self-check is working, since a self-check
doing its job means most handoffs pass on the first try. The entry follows the same shape as the
exception-rate one (what it is, why, formula, what counts, data source, cadence, how to read it
without fooling yourself, a worked synthetic example, targets). It pulls its data from the filled
checklists rather than run logs, counts only the first review so a rework cannot pad the rate, and
names its complement, first-pass approval rate, so both framings reconcile and sum to one. It stays
explicitly distinct from exception rate, a runtime signal, and it flags the honesty assumption: a
lax review makes the number look perfect and mean nothing. The intro scope line and the adding rule
were widened to cover review-record metrics, and the end lists what changed.
