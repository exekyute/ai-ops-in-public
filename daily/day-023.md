# Day 023 - Govern

**Date:** 2026-07-14
**Pillar:** Govern

## Tidbit

The failure mode Day 22 exposed is a category error, not laziness: treating "it runs" as evidence for
checks that behavior cannot answer. A working build and a broken-on-that-dimension build run identically,
so for those lines a passing run feels like a check and settles nothing. Governing against that means
making the category visible rather than telling people to try harder. So today sorts the checklist by what
actually answers each line, and names the ones only an open node, credential, or register can settle. Once
those lines are labeled, a yes on one of them off a clean run is a yes with nothing behind it. Naming the
trap does not remove it; a reviewer in a hurry can still skip. What it changes is that the skip goes from
invisible to detectable, and that is what a guardrail is for.

## Shipped

`governance/inspection-required-checks.md` (v1): the guardrail that flows from Day 22, sorting the review
checklist by what actually answers each line so "it runs" never stands in for "I looked." It names three
ways a check gets answered: the happy path shows it (does the build do the job), an adversarial run shows
it but the happy path does not (the error path, failure alerts, idempotency, the Day 015 case), and only
inspection shows it (a passing build and a broken one run identically, the Day 022 case). Then it lists the
inspection-required checks, each with its checklist line, its standard section, and exactly what to open:
secrets in the store (open the auth field), least-privilege scope (read the key's scopes), separate
test/prod creds (check the names), retries on safe steps (open the node and reason about idempotency), no
sensitive data to the model (read the payload), versioned and revertible (check the history), and one owner
named (check the register). It stays a lens on the checklist and not a replacement, is honest that the
adversarial-run checks still need their own runs, and names its ceiling: it can name the checks that demand
opening something, it cannot make anyone open them, which is exactly the blind spot the spot-audit and the
review miss rate exist for.
